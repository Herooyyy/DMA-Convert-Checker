📊 检测原理

LeechCore作弊技术流程:
┌─────────────────────────────┐
│ 副机 (作弊电脑)              │
│ • 运行作弊工具               │
│ • 连接主机LeechCore服务      │
│ • 读取主机游戏内存           │
│ • 分析坐标/敌人位置          │
│ • 实现透视作弊               │
└────────────┬────────────────┘
             │ RPC/Network通讯
             │ 大量LAN带宽
             ▼
┌─────────────────────────────┐
│ 主机 (游戏电脑)              │
│ • LeechCore服务进程          │
│ • 接收内存读取请求           │
│ • 通过驱动读取内存           │
│ • 返回内存数据               │
└────────────┬────────────────┘
             │
             ▼
    🔍 我们的检测点
    
    1. 网络检测: 100+ MB/s的LAN流量
    2. 进程检测: leechcore/memprocfs进程
    3. 服务检测: 异常的Windows服务
    4. 连接检测: 高位端口的LAN通讯
    5. 驱动检测: kmddriver等可疑驱动
    6. 内存检测: 异常的内存访问

⚙️ 配置说明

关键参数 (config.json):

| 参数 | 默认值 | 说明 |
|------|-------|------|
| detection_interval | 5 | 检测间隔(秒) |
| bandwidth_threshold_mbps | 100 | 网络异常阈值 |
| risk_threshold | 30 | 风险告警阈值 |
| enable_advanced_detection | true | 启用高级检测 |
| enable_real_time_monitoring | false | 启用实时监控 |
| monitoring_duration | 3600 | 监控时长(秒) |
| archive_enabled | true | 事件档案 |
| email_alerts_enabled | false | 邮件警报 |

调优建议:
• 减少误报: 提高bandwidth_threshold_mbps到150+
• 增加灵敏度: 降低bandwidth_threshold_mbps到50
• 性能优化: 增加detection_interval到10+
• 邮件通知: 配置email_config字段
